# 插件开发入门

## 概述

OpenTAP 插件是扩展 OpenTAP 功能的方式。一个插件可以包含 TestStep、Instrument、DUT、ResultListener、ComponentSettings 等任意组合。

## 开发环境

- **SDK**: .NET 9.0+
- **编辑器**: Visual Studio 2022 / VS Code / Rider
- **依赖**: OpenTAP NuGet 包（或直接引用 `OpenTap.dll`）

## 创建项目

```bash
# 方式 1: CLI 模板
tap sdk new project MyPlugin

# 方式 2: 手动引用 OpenTap.dll
dotnet new classlib -n MyPlugin
cd MyPlugin
dotnet add reference ../path/to/OpenTap.dll
```

## 插件类型速查

| 类型 | 基类/接口 | 用途 |
|------|----------|------|
| TestStep | `TestStep` | 测试步骤，核心功能单元 |
| Instrument | `Instrument` | 测量仪器驱动 |
| DUT | `Dut` | 被测设备驱动 |
| ResultListener | `ResultListener` | 结果收集与处理 |
| ComponentSettings | `ComponentSettings` | 全局/分组配置 |
| CliAction | `ICliAction` | CLI 命令扩展 |
| Annotator | `IAnnotator` | 自定义 Annotation |

## 最小化 TestStep

```csharp
using OpenTap;

namespace MyPlugin
{
    [Display("My Step", Group: "MyPlugin", Description: "我的第一个步骤")]
    public class MyStep : TestStep
    {
        #region Settings
        [Display("Threshold")]
        public double Threshold { get; set; } = 1.0;
        #endregion

        public override void Run()
        {
            Log.Info($"Running with threshold {Threshold}");
            UpgradeVerdict(Verdict.Pass);
        }
    }
}
```

## 打包发布

创建 `package.xml`:

```xml
<?xml version="1.0" encoding="utf-8"?>
<Package Name="MyPlugin" Version="1.0.0"
    xmlns="http://opentap.io/schemas/package">
  <Description>我的第一个插件</Description>
  <Files>
    <File Path="MyPlugin.dll" />
  </Files>
</Package>
```

```bash
# 创建包
tap package create package.xml

# 安装本地包
tap package install MyPlugin.1.0.0.TapPackage
```

## 调试

```bash
# 在 OpenTAP 安装目录中引用本地构建输出
# 或直接在源码目录运行
cd bin/Debug
./tap run mytest.TapPlan --verbose
```

## 最佳实践

1. **属性即设置** — 公开属性自动成为可配置设置
2. **用 Display 注解** — `[Display("名称", Group: "分组", Description: "描述")]`
3. **验证用 Rules** — 构造函数中 `Rules.Add(() => check, "error msg", "PropertyName")`
4. **日志用 Log** — 不用 `Console.WriteLine()`
5. **延时用 TapThread.Sleep()** — 可被中止
6. **步骤放单独文件** — 文件名接近 Display 名称
7. **接口优于具体类型** — `AllowChildrenOfType(typeof(IMyInterface))`
